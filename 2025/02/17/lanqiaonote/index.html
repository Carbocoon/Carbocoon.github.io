递归

递归实际上是函数直接或间接调用自身，通过将问题分解为更小的同类子问题来解决。

最重要的是：顺序，即如何将数据从头到尾一个不漏的按题目要求遍历一遍。

每个递归问题都可以拆解为一棵递归树，执行顺序是DFS（深度优先搜索）。

解题顺序：确定递归树 -> 写递归边界代码 -> 写一般情况的各个分支代码（注意恢复现场）

  2. 带分数
    100可以表示为带分数的形式：100 = 3 + 69258714
    还可以表示为：100 = 82 + 3546197

    注意特征：带分数中，数字1∼9分别出现且只出现一次（不包含 0）。

    类似这样的带分数，100有11种表示法。

    输入格式
    一个正整数。

    输出格式
    输出输入数字用数码 1∼9不重复不遗漏地组成带分数表示的全部种数。

    数据范围

    输入样例1：
    100
    输出样例1：
    11
    输入样例2：
    105
    输出样例2：
    6

    题解：
    之前的题都是在递归之后就解决问题的，可是递归，或者说所有算法知识点，都是题的组成部分，换言之，一道算法题里涉及到的知识点不可能只有一个（除了板子题）。在这道题中，递归就是解题中某一步骤的方式。

    现在需养成一种思维，即将题中转化为数学形式（提取题意）。例如这题，看上去很长一串，实际上翻译一下就是：n可以写成n = a + b / c，a，b，c须包含1~9中的所有数，求能写成这种形式的方法种数。

    由此，我们拆解出需要解决的问题：

    1. 使a，b，c包含1~9中所有数。
    2. 判断n = a + b / c是否成立，是就是方法总数++，不是继续。

    显而易见，第二个问题就只是一个判断的事，因此我们着重解决第一个问题。
    要使a，b，c包含1~9中所有数，既然a，b，c都是数字，则很容易想到，必须要考虑包含数字的顺序和不同的长度，与其如此，不如逆转一下思维，干脆直接求出1~9的全排列（递归）然后再遍历分出不同长度的三个数（只需分割两次）

    由此此题解决。可以看出，这个解法实际上相当暴力，但是赛场上就需要这么暴力的，因为思维简单啊。

递推(迭代)

听到递推，会很自然地想起高中数学学过数列的递推公式。例如经典的斐波那契数列，a(n) = a(n - 1) + a(n - 2) (n > 2)

重点是第一代到底干了点啥，如何对后面几代产生影响。

通过循环逐步推进问题的解决，不涉及函数自我调用。

会通过循环变量明确控制计算过程（如动态规划）

例题

  1. 简单斐波那契
    以下数列 0 1 1 2 3 5 8 13 21 … 被称为斐波纳契数列。

    这个数列从第 3 项开始，每一项都等于前两项之和。

    输入一个整数 N，请你输出这个序列的前 N 项。

    输入格式
    一个整数 N。

    输出格式
    在一行中输出斐波那契数列的前 N 项，数字之间用空格隔开。

    数据范围
    0<N<46

    输入样例：
    5
    输出样例：
    0 1 1 2 3

    题解：
    来实践一下刚才的思维。题目要求出斐波那契数列的第N项，由于数列满足a(n) = a(n - 1) + a(n - 2) (n > 2)，这样一次计算过程可以看作一次迭代，而本次迭代和下一次迭代的关系就是将三个数位置进行调换与计算。例如：0，1->1，1

    还是迭代的思路。
    第一，第一代是啥？
    第一代肯定是操作第一排拉不拉灯。这里说的操作方式肯定不止一种（题目要求6步之内出结果，那么第一排上花费了多少步骤？）
    第二，第一代对后面产生了什么影响？
    由于我们操作完第一代就接着迭代了，所以无论在第一代的操作中有没有使第一排的灯全亮了，都要进入下一代，那么要使第一排不亮的灯全亮只能依靠第二代（开一盏灯十字形改变周围状态），以此类推。

    从这道题学到的：

    1. 十进制数的二进制可以模拟对立事件的操作模式，在循环时需要位运算让最低位参与判断
    2. 如果使用一个固定次数的循环来读取5行，那么当遇到空行时，循环继续但行数索引递增，导致后续数据被错误地填充到错误的行。正确的做法是确保每个测试用例读取5个非空行，无论中间是否存在空行。然后判断特殊处理空行。
    3. 字符型的数字转化为数字型可以减’0’。
    4. 要是能想起来就剪个枝。

分治（*）

定义：将问题分解为多个相互独立的子问题，递归解决子问题后合并结果。

特点：

分解-解决-合并：子问题之间不重叠（如归并排序）。

子问题独立性：每个子问题的解可以独立计算。

更多例题详见晴问。

回溯（*）

定义：通过试探性选择路径，遇到不可行解时回退（撤销选择）的搜索算法。

特点：

试错与回退：通过深度优先搜索遍历解空间（如八皇后问题）。

状态重置：撤销当前步骤的影响，尝试其他可能性。

更多例题详见晴问。

二分

整数二分：

解题步骤：

  1. 找一个区间[L, R]，使得答案一定在这个区间中。
  2. 找一个判断条件，这个判断条件需具有二段性，且答案一定是二段性的分界点。
  3. 分析中点M在该判断条件下是否成立，考虑成立与不成立时区间更新情况。
  4. 如果更新方式写的是R = Mid， 则不用做任何处理； 如果更新方式写的是L = Mid，则需要在计算Mid时加上1。（防止死循环）

注：关于二段性，二段性是指问题的解空间可以被划分为两个性质不同的区间，且这两个区间满足单调性或明确的边界条件。简单来说，整个解空间中存在一个分界点，使得分界点左侧的所有元素满足某个条件，右侧的所有元素不满足（或相反）。这种特性使得我们可以通过二分法快速缩小解的范围，最终找到目标解。
